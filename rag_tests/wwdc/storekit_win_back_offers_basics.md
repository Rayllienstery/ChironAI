# StoreKit 2 win-back offers basics

Platform: iOS
Framework: StoreKit
MinOS: iOS 18
Difficulty: intermediate
Rag Required: no
Concept Mode: all
RAG Strict: true

## Question

In the WWDC24 StoreKit session Apple introduced **win-back offers** for subscriptions.  
Explain:

- what a win-back offer is and how it differs from introductory and promotional offers;
- which customers are eligible (churned subscribers) and how eligibility is configured in App Store Connect;
- how you can merchandise win-back offers in your app using the StoreKit Message API without writing additional custom UI code;
- how these offers can also be promoted on the App Store (for example, being featured by editorial teams).

You do **not** need to provide full code, but outline the end-to-end flow from configuring a win-back offer in App Store Connect to showing it in-app using StoreKit 2.

## Expected Concepts

- win-back offers
- churned subscribers
- eligibility rules
- App Store Connect
- StoreKit Message API
- subscription offer types

## RAG Requirement

The answer should be based on the WWDC24 \"Implement App Store Offers\" / StoreKit sessions, not just generic StoreKit documentation.
