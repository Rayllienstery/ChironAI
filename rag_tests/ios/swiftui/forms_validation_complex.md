# Complex forms and validation in SwiftUI

Platform: iOS
Framework: SwiftUI
MinOS: iOS 16
Difficulty: intermediate
Concept Mode: all
RAG Strict: false

## Question

You build a multi-step form in SwiftUI that collects user profile data across several screens.
Explain:

- how to keep a single source of truth for the form model across steps;
- how to perform per-field and cross-field validation and reflect errors in the UI;
- how to enable/disable navigation or submit buttons based on validation state;
- common mistakes (duplicating form state, validating only on submit, mixing view and model concerns).

## Expected Concepts

- single source of truth
- validation
- error state
- multi-step form

## RAG Requirement

The answer should align with SwiftUI documentation and best practices for state management in forms.

Checks Senior-level understanding of building robust forms in SwiftUI.
